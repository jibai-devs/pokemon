## Summary

Build `scripts/featurize.py` that converts Kaggle replay JSON files into numpy arrays ready for training.

## Output format

Per decision step, three arrays:
- `state` — float32, shape `(~434,)` — board state encoding
- `options` — float32, shape `(N_opts, 64)` — per-option feature vectors (variable N, padded)
- `label` — int64, shape `(max_count,)` — chosen option indices

Stored as `.npz` per game, or concatenated into a single dataset file in `data/replays/processed/`.

## State vector (~434d)

See `docs/plans/001_training_pipeline.md` §4a for the full breakdown. Key fields:
- Turn info (4): turn/100, supporterPlayed, energyAttached, retreated
- My active (35): card_id_emb(32) + hp_ratio + energy_count/10 + has_tool
- My bench ×5 (175): same per slot, zero-pad empty
- My counts (3): handCount/20, deckCount/20, prizes_left/6
- My status flags (5): poisoned/burned/asleep/paralyzed/confused
- Opp active (34): card_id_emb + hp_ratio + energy_count/10
- Opp bench ×5 (170): same
- Opp counts (3)
- Opp status (5)

Card ID embedding: looked up from a pre-built `card_id → int` index (IDs from `all_cards.json`). Embedding layer trained end-to-end.

## Option vector (64d)

One-hot option type (15d) + context (15d) + card/attack embedding (32d) + misc (2d).

Option types: Play, Attach, Evolve, Retreat, Attack, End, Card, Yes, No (from Kaggle string enums).
Context: Main, ToHand, AttachFrom, AttachTo, SetupActivePokemon, Switch, ToActive, Discard, Activate, Mulligan, IsFirst.

For `Attack` options: use `attackId → attack_emb(32)`.
For `Play/Card` options: use resolved card_id → card_emb(32).

## Notes

- In Kaggle replay data, opponent hand IS visible — featurize it fully. Mask at inference.
- `selected` normalisation (PKM-006) must be resolved first — featurizer assumes option-list indices
- Card/attack embeddings are indices into `nn.Embedding` layers, not the raw IDs — build a `card_id → embedding_index` vocab map
