## Summary

In the Kaggle replay format, the `selected` field in `Card/*` contexts may encode the card's area-index rather than the option-list index. This needs to be verified and normalised before featurization.

## Evidence

Step 2 of `example_replay.json` (`Card/SetupActivePokemon`):
- Only 1 option in the list (index 0): `CARD Relicanth (p0 hand[0])`
- `selected = [1]` — does not match option-list index 0

Step 5 (`Card/ToHand`):
- 4 options (indices 0–3), `selected = [0]` → correctly picks option 0 (Relicanth)

`Main/Main` context (63/112 steps): alignment appears correct throughout.

## Hypothesis

For `Card/*` contexts, `selected` may contain one of:
- The card's index within its area (hand[1], bench[0], etc.)
- The card's serial number

Need to check a `Card/SetupActivePokemon` frame more carefully — compare `selected` value against all card serials in the hand to see if it's a serial lookup.

## How to investigate

```python
# In parse_replay.py or a notebook:
v2 = vis[2]  # SetupActivePokemon frame
print(v2['selected'])          # [1]
print(v2['select']['option'])  # [{'area':2, 'index':0, ...}]
players = v2['current']['players']
hand = players[0]['hand']
for i, c in enumerate(hand):
    print(i, c.get('serial'), c.get('name'))
# Does serial==1 match Relicanth?
```

## Root cause (confirmed)

`selected` encodes different things depending on context:
- `Main/Main`, `YesNo/*`, most `Card/*`: **option-list index** (correct)
- `Card/Switch`, `Card/SetupActivePokemon` (mulligan path): selected = promoted/placed card's serial (not in board state because card already moved)
- `Card/ToHand` (Night Stretcher): selected = serial of retrieved card from discard/bench
- `Card/AttachFrom`: selected has one extra value beyond n_opts

Affects ~11/112 frames (~10%) in one replay. All strategically minor (which energy to pick, which bench mon to promote).

## Fix (applied)

`_extract_kaggle()` now validates each sample: `valid = all(0 <= sel < n_opts for sel in selected)`. Invalid samples are included in the output list but flagged with `valid=False`. The display marks them with `!! INVALID selected (skipped in training) !!`. The summary line reports `Valid: N | Invalid: M`.

The featurizer (PKM-008) must filter on `sample["valid"] == True`.
