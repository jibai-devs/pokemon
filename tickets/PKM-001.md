## Summary

The competition requires a `deck.csv` file at the top level of the submission `.tar.gz`. It must contain exactly 60 card IDs, one per line. This is how the platform knows which deck to play — it is read separately from the agent function.

## Acceptance criteria

- `deck.csv` exists at the repo root
- Contains exactly 60 lines, each a single integer card ID
- IDs match the current Fire deck (`FIRE_DECK` in `src/pokemon/decks.py`)

## Notes

- Current deck: 2× Gouging Fire ex (46), 4× Slugma (76), 4× Magcargo ex (30), trainers, 33× Fire Energy (2)
- Order within the file does not matter — the engine shuffles the deck before play
- When the deck changes, this file must be regenerated to match
