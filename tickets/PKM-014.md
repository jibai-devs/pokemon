## Summary

Add a Monte Carlo Tree Search layer that runs at inference time on top of the
BC (and later PPO) policy/value network, instead of just taking the network's
top-scoring option directly. This is the AlphaZero-style recipe: policy net
proposes which options are worth exploring, value net judges resulting
positions, MCTS searches a few plies ahead using the engine as a forward
model, and tallies which root option led to the best simulated outcomes.

## Motivation

BC/PPO scores options in one shot per decision. MCTS lets the agent "spend
more thinking time" on consequential decisions (attack sequencing, energy
attachment, retreat timing) by simulating forward a few moves, which should
raise the ceiling beyond the raw policy — and is a hedge if PPO training
(PKM-010) doesn't converge to something strong by the deadline: BC + MCTS
alone should still beat BC-argmax.

## Approach

- Requires a forward model to simulate "what happens if I play this move" —
  use the CABT engine (`libcg.so` via WSL) or a local simulator
- Use policy network output to prioritize which candidate moves to expand
  (avoid wasting simulations on obviously bad options)
- Use value network output to score leaf/cutoff positions instead of always
  rolling out to game end
- Aggregate visit counts / average outcomes across simulations to pick the
  actual move to play
- Opponent's hidden hand breaks the "full information" assumption classic
  MCTS relies on — see PKM-015 (determinization / belief modeling) for how
  to handle this. A naive first pass can ignore opponent hidden info and
  lean on the value network to absorb that uncertainty.

## Success criterion

BC + MCTS agent beats BC-argmax-only agent in head-to-head evaluation.

## Notes

- Decision-time-only addition — does not require retraining BC/PPO, but
  benefits from PKM-010's stronger value head once available
- Simulation budget (sims per move) is constrained by engine speed in WSL —
  profile `libcg.so` step latency before committing to a per-move sim count
- Sequence after PKM-009 (need a BC policy + value head to search with);
  can run independent of / in parallel with PKM-010
