## Summary

Build `scripts/parse_replay.py` to parse Kaggle episode replay JSON files and print each decision step in human-readable form.

## What was built

Handles two formats:

**Kaggle format (ver=2)** — downloaded via `kaggle competitions replay <id>`:
- String enums: `opt.type='Play'/'Attack'/'End'/...`, `select.type='Main'/'Card'/'YesNo'`
- `selected` field holds chosen option indices (training label)
- Both players' hands fully visible in replay data
- Cards carry inline `name` field — no catalog lookup needed
- `current` board state at top level of each vis frame

**Local format** — from `env.run()` via `--local` (requires WSL):
- Integer enums matching the live engine API
- `action` field holds `[p0_action, p1_action]`; active player from `current.yourIndex`

## Verified on

`example_replay.json` — ShumpeiNomura vs Takaaki Matsuda, 112 decision steps, Metal deck (Relicanth + Duraludon + Archaludon ex). Parser detected kaggle format, resolved card names inline, showed both hands, correct winner in summary.

## Known issue

See PKM-006 — `selected` for `Card/*` contexts may encode area-index rather than option-list index. Affects ~49/112 steps. `Main/Main` steps (63/112, the strategically important ones) are correct.
