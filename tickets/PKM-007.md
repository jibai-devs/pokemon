## Summary

Bulk-download episode replays from top-rated leaderboard agents to use as behavioral cloning training data.

## Resolution

Found a better path than scraping individual leaderboard submissions: Kaggle
publishes a daily dataset (`kaggle/pokemon-tcg-ai-battle-episodes-<date>`) for
each of the last 15 days, and every dataset ships a `manifest.csv` with one
row per episode (`episode_id, avg_score, min_score, sum_score, agent_count,
size_bytes`). `avg_score` is the agents' rating for that game, so we can pick
the top-N highest-rated episodes per day and download only those files
instead of the full multi-GB daily archive.

Wrote `scripts/download_replays.py`:
- Pulls `manifest.csv` for each of the 15 available days
- Sorts by `avg_score` descending, takes the top 100 episodes/day
- Downloads just those episode JSON files via the Kaggle API (`dataset_download_file`), skipping any already present (resumable)

Ran it for all 15 days x top 100 → **1,500 replays, 4.6GB**, stored in
`data/replays/raw/`. Verified format with `scripts/parse_replay.py` on a
sample — parses cleanly as `ver=2` kaggle format (string enums, `selected`
labels, both hands visible), 162 steps / 131 valid training samples in the
sample file.

## Target

At least 500 replays from competitive agents (~50,000+ decision steps). **Exceeded: 1,500 replays downloaded.**

## Notes

- Confirmed: Kaggle replays use `ver=2` format with string enums and `selected` field
- Both players' hands are visible in replay data — more signal than live observations
- Manifest-based rating filter made the "top-rated agents" requirement much cheaper than per-submission scraping — no leaderboard browsing needed
- Raw JSON stored in `data/replays/raw/`, per-day manifests cached in `data/replays/manifests/`
- Re-run `scripts/download_replays.py --top-per-day N` anytime to pull more (it skips existing files)
- Also note which decks appear frequently — feeds into PKM-011 (deck evaluation) and PKM-012 (meta analysis)
