## Summary

The user will provide a CSV of all Pokemon TCG Pocket cards. Process it into:
1. A filtered, enriched dataset covering only the competition card pool (1267 cards from `all_cards.json`)
2. Derived features useful for analytics (deck building, meta analysis) and ML (featurization)

## Input

User-provided CSV at `data/cards_raw.csv` (format TBD once file is added).

## Output

`data/cards_processed.csv` — one row per card, with columns:

| Column | Description |
|--------|-------------|
| `card_id` | Engine card ID (matches deck lists and replay data) |
| `name` | Card name |
| `card_type` | `pokemon` / `trainer` / `energy` |
| `pokemon_type` | Fire / Water / Metal / etc. (null for non-pokemon) |
| `stage` | Basic / Stage1 / Stage2 / ex (null for non-pokemon) |
| `hp` | Hit points (null for non-pokemon) |
| `retreat_cost` | Energy required to retreat |
| `atk1_id` | First attack ID |
| `atk1_name` | First attack name |
| `atk1_damage` | First attack base damage |
| `atk1_cost` | First attack energy cost (total) |
| `atk2_id` | Second attack ID (null if none) |
| `atk2_name` | ... |
| `atk2_damage` | ... |
| `atk2_cost` | ... |
| `damage_per_energy` | Max damage / total energy cost (efficiency metric) |
| `in_competition_pool` | Bool — whether this card ID appears in `all_cards.json` |

## Processing steps

1. Load user CSV, normalise column names
2. Join on card name or ID with `reverse-engineering/data/all_cards.json` to get engine `card_id`
3. Join with `reverse-engineering/data/all_attacks.json` for attack stats
4. Derive `damage_per_energy`, `stage`, `in_competition_pool`
5. Filter to `in_competition_pool == True` for the working dataset
6. Write `data/cards_processed.csv`

## Analytics this enables

- **Deck building:** sort by `damage_per_energy` to find efficient attackers per type
- **Meta analysis (PKM-012):** label archetype fingerprints with card names and types
- **Featurization (PKM-008):** richer card feature vector beyond just embedding index — add HP, type, stage as explicit features
- **Report:** card choice rationale section (20% of Strategy score) — justify deck picks with efficiency stats

## Notes

- Card ID join may need fuzzy name matching if the CSV uses different card names than the engine
- Energy cards and basic trainers have simpler schemas — handle separately
- Once processed, `data/cards_processed.csv` becomes the canonical card reference for all scripts
