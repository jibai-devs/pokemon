## Resolution

Refactored `pokemon.decks` into a registry (`DECKS: dict[str, list[int]]`,
`ACTIVE_DECK_NAME` / `ACTIVE_DECK`) instead of a single hardcoded
`FIRE_DECK`. `pokemon.agent.fire_agent` was replaced with
`make_agent(deck)` (deck-agnostic) plus `default_agent` bound to
`ACTIVE_DECK`. `cli.py` gained `-d/--deck` on `play` and a new
`export-deck` command that regenerates `deck.csv` from the registry (it
was previously hand-written and could drift from `pokemon.decks`).

Added `PSYCHIC_DECK` (60 cards ŌĆö Mega Kangaskhan ex / Latias ex shell,
SlowpokeŌåÆSlowking, Metagross, Ultra Ball/Poke Pad/Ciphermaniac's
Codebreaking engine, Academy at Night, Psychic energy line) as supplied
by the user, set `ACTIVE_DECK_NAME = "psychic"`, added
`deck/001_psychic_deck.py` + `.md` artifacts, and regenerated `deck.csv`.
Added display-name overrides in `catalog.py` for the new card IDs.

Verified: deck sums to 60, all card names resolve, `deck.csv` matches
`PSYCHIC_DECK`. Not yet run through an actual game (needs WSL /
`kaggle_environments`, unavailable in the dev shell used).

## Summary

The fire deck was hardcoded in multiple places (`decks.py`, `agent.py`'s
`fire_agent`, hand-written `deck.csv`) with no single point to swap decks.
Needed a registry so switching the competition deck to Psychic (and any
future deck) is a one-file edit.
