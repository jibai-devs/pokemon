## Summary

Use verbose game logs as the feedback signal for improving
`DEFAULT_PSYCHIC_HEURISTICS`, instead of hand-picking the next heuristic to
add/fix from intuition alone. An agent (or a script an agent drives) reads
a batch of `play -a heuristic -g N -v` traces, identifies concrete
recurring mistakes (a heuristic firing when it shouldn't, a decision point
where no heuristic fires and the random fallback picks something bad, a
resource spent on the wrong target), and proposes/implements a specific
heuristic change in response.

## Motivation

The Seek Inspiration bug fixed in PKM-017 (2026-07-05) is the model for
this: reading a live WSL playtest trace showed `prefer_copy_fodder_targets`
ranking Metagross/Kyurem ahead of Slowking/Slowpoke for *every* search
context, including `TO_HAND` — which meant Ultra Ball/Poke Pad kept
spending themselves pulling Seek Inspiration's discard fodder out of the
deck and into hand, where it was useless. That fix came from eyeballing
one verbose trace by hand. That doesn't scale as more heuristics get
added — want a repeatable process (loop, not one-off audit) for turning
"the agent lost, here's the log" into "here's the specific heuristic gap
and a proposed fix."

## Approach (subject to revision)

1. Run a batch of verbose games (`play -a heuristic -g N -v`) and save logs
   — probably to `data/` or a scratch dir, not committed.
2. Fix the known cosmetic bug first (`catalog.format_option` mislabels
   bench/active/deck-area options — see AGENTS.md "Known issues" and
   PKM-017): reading logs by eye to find errors is unreliable while option
   labels can be wrong. Low effort, high leverage for this ticket
   specifically.
3. For losses (and close/long games), identify:
   - Points where `fallback random` fired and the random pick was
     clearly bad (e.g. discarding a key piece, attaching energy to the
     wrong Pokemon, retreating into a worse attacker)
   - Points where an existing heuristic fired but produced a bad outcome
     (wrong priority ordering, a heuristic firing in a context it
     shouldn't, e.g. the PKM-017 bug)
   - Recurring patterns across multiple games, not one-off variance
4. Turn each finding into a proposed heuristic change (new rule, reordered
   priority, tightened context gate) with the specific log evidence cited
   — same evidence bar as the PKM-017 fix (quote the actual option/attack
   ids and context, not just "seemed suboptimal").
5. Implement, add/adjust unit tests in `tests/test_heuristics.py`, then
   re-run a batch and compare win-rate before/after (same n, same
   opponent) to confirm the change actually helped before keeping it.

## Open questions

- How many games are enough to trust a signal over noise? PKM-017's 20-game
  batches (0% -> 25%) are a reasonable current baseline for "believable
  swing," but worth deciding a minimum n before calling a change confirmed.
- Should this become a script (`scripts/analyze_heuristic_logs.py`) that
  an agent runs and summarizes, or stay a fully manual "run it, read it,
  fix it" loop for now? Leaning toward starting manual and only automating
  once the same log-reading steps get repeated a few times.
- Worth capturing *why* a heuristic fired (not just that it fired) in
  verbose output, to make this analysis less dependent on reconstructing
  intent from raw option dicts each time.

## Depends on
